🚀 Antigravity 기반 '바이브 코딩' 원고 게시 마스터 플랜

1. 파일 포맷은 무엇으로 저장해야 할까?

결론: 무조건 Markdown (.md) 파일로 저장하고 관리해야 합니다.

이유 1: AI (Antigravity) 친화적
pdf나 doc, hwp 파일은 디자인 서식이 섞여 있어 AI 에이전트가 코드를 읽거나 텍스트를 정확하게 자르기 힘듭니다. 반면, .md 파일은 순수 텍스트 기반이라 Antigravity의 에이전트가 완벽하게 읽고, 수정하고, 여러 파일로 분할할 수 있습니다.

이유 2: 코드 블록 지원
원고 내용에 터미널 명령어, 파이썬 코드, 설정 파일 내용이 많습니다. 마크다운은 ```python 처럼 코드를 예쁘게 담는 데 특화된 포맷입니다.

이유 3: 웹 플랫폼 호환성
워드프레스는 물론이고, 최신 블로그 시스템(Next.js, Astro 등) 모두 마크다운 포맷을 기본 게시물 포맷으로 사용합니다.

2. 🖼️ 그림(이미지) 파일은 어떻게 처리할까? (핵심)

마크다운은 텍스트로만 이루어져 있으므로, 이미지는 별도의 폴더에 저장한 뒤 다음과 같은 문법으로 '링크'를 걸어 보여줍니다.

문법: ![이미지에 대한 설명](이미지_파일_경로)

예시: ![PostgreSQL DB 연동 화면](/images/db-setting.png)

💡 실전 이미지 추출 및 관리 꿀팁

현재 구글 문서나 워드에 텍스트와 이미지가 섞여 있다면, 이미지를 하나씩 우클릭해서 저장하기 매우 힘듭니다.

일괄 추출: 구글 문서(또는 워드)에서 파일 > 다운로드 > 웹페이지(HTML, 압축됨)를 선택합니다.

다운로드된 압축 파일을 풀면 images라는 폴더 안에 문서에 쓰인 모든 그림 파일이 순서대로 예쁘게 모여 있습니다.

이 images 폴더를 Antigravity 작업 폴더 안으로 복사해서 가져옵니다.

🌐 배포 방식에 따른 이미지 경로 설정법

자체 블로그(Next.js 등) 구축 시: Antigravity 프로젝트 폴더 내 /public/images/ 위치에 그림을 모두 넣으면 끝납니다. 마크다운에서는 ![설명](/images/파일명.png)으로 작성하면 알아서 이미지가 뜹니다.

워드프레스 업로드 시: 이미지는 텍스트와 달리 직접 워드프레스의 '미디어 라이브러리'에 업로드해야 합니다. 또는 이미지만 모아서 구글 드라이브나 Imgur 같은 외부 호스팅에 올린 후, 그 URL 주소(https://.../chart.png)를 마크다운에 적어주어야 합니다.

3. Antigravity를 활용한 체계적 게시 워크플로우

Antigravity의 AI 에이전트를 활용하면, 수작업으로 복사/붙여넣기 할 필요 없이 원고 분할부터 업로드까지 반자동화할 수 있습니다.

단계 1: 워크스페이스 세팅 및 원본 변환

Antigravity 실행: 새로운 프로젝트 폴더를 만들고 Antigravity로 엽니다. (예: vibecoding-blog)

원본 업로드: 텍스트 원본은 original_manuscript.md 로 저장하고, 추출한 이미지들은 /public/images/ 폴더에 넣습니다.

단계 2: AI 에이전트를 통한 자동 챕터 분할 (프롬프트 활용)

Antigravity의 Agent 창을 열고, 직접 자르는 대신 에이전트에게 지시합니다.

💡 Antigravity 프롬프트 예시:
"@workspace 에 있는 original_manuscript.md 파일을 읽고, 워드프레스에 연재하기 좋도록 목차의 '소제목(예: 10-1, 11-2)'을 기준으로 파일을 분할해줘. 분할된 파일은 /content/posts/ 폴더에 각각 저장하고, 파일명은 10-1-chart-advanced.md처럼 영어 소문자와 하이픈으로 만들어줘. 그리고 원본에 있던 이미지 파일명과 매칭해서 ![설명](/images/이미지명.png) 형태로 이미지 링크도 잘 살려줘."

결과물 (자동 생성된 파일 구조):

/content
  /posts
    ├── 00-intro.md
    ├── 10-1-chart-advanced.md
    └── 11-1-redis-caching.md
/public
  /images
    ├── chart1.png
    └── console_capture.png


단계 3: SEO 및 메타데이터(Frontmatter) 자동 삽입

💡 Antigravity 프롬프트 예시:
"방금 생성한 /content/posts/ 안의 모든 마크다운 파일 최상단에 Frontmatter(메타데이터)를 추가해줘.
(title, category, tags, date, summary 포함)"

단계 4: 웹 게시 방식 결정 및 자동화

이 단계에서 기존 도메인(firemarkets.net)의 서브도메인을 활용하여 배포합니다.

옵션 A: 기존 워드프레스에 REST API로 자동 발행

워드프레스 관리자에서 '응용 프로그램 비밀번호'를 발급받습니다.

에이전트에게 지시: "이 마크다운 파일들과 이미지들을 워드프레스 REST API를 사용해서 '임시저장' 상태로 일괄 업로드하는 파이썬 스크립트를 짜고 실행해줘."

스크립트가 실행되면 워드프레스에 임시저장 되며, 확인 후 '발행'만 누르면 됩니다.

옵션 B: Antigravity로 개발 블로그 자체 구축 (추천)

"바이브 코딩" 주제에 가장 잘 맞고 마크다운 파일과 이미지를 가장 쉽게 렌더링하는 방법입니다.

에이전트에게 지시: "Next.js와 Tailwind CSS를 사용해서 이 폴더의 마크다운 파일과 /public/images/의 이미지를 렌더링하는 기술 블로그를 만들어줘."

코드가 완성되면 Vercel 등에 배포하고 vibecoding.firemarkets.net 도메인을 연결합니다.

🎯 최종 행동 지침 요약

포맷 변환: 텍스트는 마크다운(.md) 파일 하나로 합칩니다.

에이전트 지시: Antigravity 에이전트를 통해 소제목 단위로 파일을 쪼개고 이미지 링크(![설명](/images/파일명))를 자동 복원시킵니다.

배포: 생성된 파일 뭉치를 바탕으로 자체 블로그(Next.js)를 배포하거나 워드프레스에 연동합니다.
